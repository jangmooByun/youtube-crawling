## 중단된 작업 — 다음 사이클에서 재개해야 할 항목

### 1. crawl-links 미처리 큐
- **`sources.fetched_at IS NULL` 이고 IG/TT 제외 = 166 sources** (Round 2 priority 큐 중 4개만 처리됨)
- 채널당 우선순위 1개씩만 남아있는 상태 (LIB → personal 순). 그대로 다시 `crawl-links` 돌리면 됨.
- 주의: 1차 사이클에선 165→1.5시간이었는데 2차에선 4개에서 매우 느려짐. **첫 사이트들이 응답이 느린지 / 평균 사이트 지연이 늘어났는지** 진단 필요.

### 2. deferred sources (낮은 우선순위)
- **`sources.fetched_at = '2099-01-01 00:00:00'` 인 sources = 1,760개**.
- 채널당 2번째 이상 외부링크들. SQL 한 번에 큐 다시 살리려면:
  ```sql
  UPDATE sources SET fetched_at = NULL, success = NULL
  WHERE fetched_at = '2099-01-01 00:00:00';
  ```
- 단 이걸 다 돌리면 ~12시간 이상 걸림. 채널당 우선순위 2-3개씩으로 다시 제한 권장 (`scripts/enrich_from_videos_2026-04-30.py` 의 SQL 주석 참고).
